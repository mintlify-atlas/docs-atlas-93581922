The Gleam CLI is the primary tool for creating, building, testing, and publishing Gleam projects. All commands are invoked as `gleam <command>`.

## Getting help

```bash
# Show top-level help
gleam help

# Show help for a specific command
gleam <command> --help

# Print the installed Gleam version
gleam --version
```

## Full command reference

| Command | Description |
|---|---|
| `gleam new` | Create a new project |
| `gleam build` | Compile the project |
| `gleam check` | Type-check the project without code generation |
| `gleam run` | Run the project's `main` function |
| `gleam dev` | Run the project's `<package>_dev` module |
| `gleam test` | Run the project's test suite |
| `gleam format` | Format source code |
| `gleam fix` | Rewrite deprecated Gleam code |
| `gleam shell` | Start an Erlang REPL with Gleam code loaded |
| `gleam add` | Add new dependencies to `gleam.toml` |
| `gleam remove` | Remove dependencies from `gleam.toml` |
| `gleam update` | Update dependencies to their latest compatible versions |
| `gleam deps` | Subcommands for inspecting dependencies |
| `gleam publish` | Publish the package to Hex.pm |
| `gleam hex` | Subcommands for interacting with Hex |
| `gleam docs` | Build or publish HTML documentation |
| `gleam export` | Export project artifacts |
| `gleam clean` | Delete build artifacts |
| `gleam lsp` | Start the language server (used by editors) |

## Global behavior

- Commands that build code automatically download missing dependencies before compiling.
- Resolved dependency versions are locked in `manifest.toml`. Do not edit this file by hand.
- The `--target` flag is available on build, run, test, check, and docs commands. When omitted the target defaults to `erlang`.
- Progress output is printed to stderr. Pass `--no-print-progress` to suppress it where supported.
- Logging can be enabled by setting the `GLEAM_LOG` environment variable (e.g. `GLEAM_LOG=info`).
- Color in log output can be disabled by setting `GLEAM_LOG_NOCOLOUR`.
