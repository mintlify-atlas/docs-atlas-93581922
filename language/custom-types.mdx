title: "Custom types"
description: "Defining custom types, records, enums, generics, opaque types, and recursive types in Gleam."
---

Custom types are the primary way to model your domain in Gleam. They can have one or more variants, carry labelled or unlabelled fields, be generic, and be opaque.

## Defining a custom type

Use `pub type` to define a public custom type (or `type` for a module-private type). Each variant is a constructor function.

```gleam
pub type Direction {
  North
  South
  East
  West
}
```

Constructors are values and can be used anywhere a value is expected.

## Single-variant types (record-like)

A type with one variant acts like a record or struct:

```gleam
pub type User {
  User(name: String, age: Int, email: String)
}
```

Create a value by calling the constructor:

```gleam
let user = User(name: "Alice", age: 30, email: "alice@example.com")

// Positional (label-free) construction also works
let user2 = User("Bob", 25, "bob@example.com")
```

### Accessing fields

Use `.field_name` to access a labelled field:

```gleam
let name = user.name   // "Alice"
let age = user.age     // 30
```

<Note>
Field access only works on labelled fields. Unlabelled fields can only be accessed through pattern matching.
</Note>

## Multi-variant types (enum-like)

A type with multiple variants is like an enum where each variant can carry different data:

```gleam
pub type Shape {
  Circle(radius: Float)
  Rectangle(width: Float, height: Float)
  Point
}

pub fn area(shape: Shape) -> Float {
  case shape {
    Circle(radius: r) -> 3.14159 *. r *. r
    Rectangle(width: w, height: h) -> w *. h
    Point -> 0.0
  }
}
```

## Labelled fields

Labels document the meaning of each field at construction and access sites. The label and the internal variable name can differ:

```gleam
pub type Connection {
  Connection(host host: String, port port: Int)
}

let conn = Connection(host: "localhost", port: 5432)
let h = conn.host  // "localhost"
```

<Tip>
Use labelled fields whenever a type has multiple fields of the same type, or when the meaning of a field is not obvious from its type.
</Tip>

### Mixed labelled and unlabelled fields

A type can mix labelled and unlabelled fields. Unlabelled fields must appear before labelled ones.

```gleam
pub type Packet {
  Packet(Int, Float, label_a: Int, label_b: String)
}

let p = Packet(1, 3.14, label_a: 42, label_b: "hello")
```

## Record update syntax

Create a copy of a record with some fields replaced using `..`:

```gleam
pub type Person {
  Person(name: String, age: Int, country: String)
}

pub fn have_birthday(person: Person) -> Person {
  Person(..person, age: person.age + 1)
}

pub fn move_to(person: Person, new_country: String) -> Person {
  Person(..person, country: new_country)
}
```

The original record is not modified; a new value with the updated fields is returned.

```gleam
let alice = Person(name: "Alice", age: 30, country: "UK")
let older_alice = Person(..alice, age: 31)
// alice.age == 30  (unchanged)
// older_alice.age == 31
```

## Generic custom types

Type parameters make a custom type generic. They are lowercase names inside parentheses after the type name.

```gleam
pub type Box(a) {
  Box(tag: String, value: a)
}

pub type Either(a, b) {
  Left(a)
  Right(b)
}
```

Usage:

```gleam
let int_box = Box(tag: "count", value: 42)
let str_box = Box(tag: "name", value: "Alice")

let result: Either(String, Int) = Left("hello")
```

The standard library `Result` and `Option` types are both generic custom types:

```gleam
// Defined in the prelude — shown here for illustration
pub type Result(value, error) {
  Ok(value)
  Error(error)
}

pub type Option(a) {
  Some(a)
  None
}
```

## Opaque types

Mark a type as `pub opaque type` to expose it publicly but hide its constructors and field access. Callers can hold values of the type and pass them to your module's functions, but cannot construct or inspect them directly.

```gleam
pub opaque type Queue(a) {
  Queue(front: List(a), back: List(a))
}

pub fn empty() -> Queue(a) {
  Queue(front: [], back: [])
}

pub fn push(queue: Queue(a), item: a) -> Queue(a) {
  Queue(..queue, back: [item, ..queue.back])
}

pub fn pop(queue: Queue(a)) -> Result(#(a, Queue(a)), Nil) {
  case queue.front {
    [head, ..tail] -> Ok(#(head, Queue(..queue, front: tail)))
    [] ->
      case list.reverse(queue.back) {
        [] -> Error(Nil)
        [head, ..tail] -> Ok(#(head, Queue(front: tail, back: [])))
      }
  }
}
```

Outside this module, `Queue` can be used but not constructed or destructured:

```gleam
// This is fine:
let q = queue.empty()
let q = queue.push(q, "hello")

// This is a compile error — Queue constructor is not accessible:
// let q = Queue(front: [], back: [])
```

## Recursive types

A custom type can refer to itself. This is the standard way to define tree or linked-list structures.

### Linked list

```gleam
pub type List(a) {
  Empty
  Node(value: a, next: List(a))
}

pub fn length(list: List(a)) -> Int {
  case list {
    Empty -> 0
    Node(_, next) -> 1 + length(next)
  }
}
```

### Binary tree

```gleam
pub type Tree(a) {
  Leaf
  Branch(value: a, left: Tree(a), right: Tree(a))
}

pub fn insert(tree: Tree(Int), value: Int) -> Tree(Int) {
  case tree {
    Leaf -> Branch(value: value, left: Leaf, right: Leaf)
    Branch(value: v, left: l, right: r) if value < v ->
      Branch(value: v, left: insert(l, value), right: r)
    Branch(value: v, left: l, right: r) ->
      Branch(value: v, left: l, right: insert(r, value))
  }
}
```

## Constants from custom types

Constants can use custom type constructors as long as the arguments are also constants:

```gleam
pub type Direction {
  North
  South
  East
  West
}

pub const default_direction = North
```

## Deprecating variants

Individual variants can be marked as deprecated:

```gleam
pub type HashAlgorithm {
  Sha256
  Sha512
  @deprecated("MD5 is insecure — use Sha256")
  Md5
}
```

The compiler emits a warning when the deprecated variant is used.
