---
title: "Configuration"
description: "Reference for gleam.toml configuration options"
---

Every Gleam project has a `gleam.toml` file at its root. It defines the package name, version, dependencies, and build settings.

## Complete example

```toml gleam.toml
name = "my_app"
version = "1.0.0"
gleam = ">= 1.0.0"
description = "A Gleam application"
licences = ["Apache-2.0"]
target = "erlang"
internal_modules = ["my_app/internal", "my_app/internal/*"]

[repository]
type = "github"
user = "my-org"
repo = "my_app"

[[links]]
title = "Website"
href = "https://example.com"

[documentation]
pages = [
  { title = "Getting started", path = "getting-started.html", source = "docs/getting-started.md" }
]

[dependencies]
gleam_stdlib = ">= 0.70.0 and < 2.0.0"

[dev_dependencies]
gleeunit = ">= 1.0.0 and < 2.0.0"

[erlang]
extra_applications = ["inets", "ssl"]
application_start_module = "my_app"

[javascript]
typescript_declarations = true
runtime = "node"
```

## Field reference

<AccordionGroup>
  <Accordion title="name">
    **Required.** The package name. Must be `snake_case` and unique on Hex if you plan to publish.

    ```toml
    name = "my_app"
    ```
  </Accordion>

  <Accordion title="version">
    A [semantic version](https://semver.org) string. Defaults to `"0.1.0"` when generated by `gleam new`.

    ```toml
    version = "1.0.0"
    ```
  </Accordion>

  <Accordion title="gleam">
    Minimum Gleam compiler version required to build this package. Expressed as a Hex version requirement string.

    ```toml
    gleam = ">= 1.0.0"
    ```

    <Note>
      Running `gleam build` with an incompatible compiler version produces an error.
    </Note>
  </Accordion>

  <Accordion title="description">
    A short human-readable description of the package. Displayed on Hex and in generated documentation.

    ```toml
    description = "A Gleam web framework"
    ```
  </Accordion>

  <Accordion title="licences">
    An array of [SPDX license identifiers](https://spdx.org/licenses/). Required when publishing to Hex.

    ```toml
    licences = ["Apache-2.0"]
    # or multiple
    licences = ["Apache-2.0", "MIT"]
    ```
  </Accordion>

  <Accordion title="target">
    The default compilation target. Accepted values are `"erlang"` (default) and `"javascript"`. This value is used when no `--target` flag is passed to CLI commands.

    ```toml
    target = "javascript"
    ```

    See [Compilation targets](/project/targets) for full details.
  </Accordion>

  <Accordion title="internal_modules">
    Glob patterns for modules that should be excluded from generated HTML documentation. Defaults to `["<name>/internal", "<name>/internal/*"]` when omitted.

    ```toml
    internal_modules = ["my_app/internal", "my_app/internal/*"]
    ```
  </Accordion>

  <Accordion title="[dependencies]">
    Production dependencies resolved from Hex. Each key is a package name; the value is a Hex version requirement string.

    ```toml
    [dependencies]
    gleam_stdlib = ">= 0.70.0 and < 2.0.0"
    gleam_http = ">= 3.0.0 and < 4.0.0"
    ```
  </Accordion>

  <Accordion title="[dev_dependencies]">
    Dependencies used only during development and testing. Not included in production builds.

    ```toml
    [dev_dependencies]
    gleeunit = ">= 1.0.0 and < 2.0.0"
    ```
  </Accordion>

  <Accordion title="[repository]">
    Metadata about the source repository. Used to generate links in HTML documentation. Supported `type` values: `"github"`, `"gitlab"`, `"bitbucket"`, `"codeberg"`, `"sourcehut"`, `"gitea"`, `"forgejo"`, `"tangled"`, `"custom"`.

    ```toml
    [repository]
    type = "github"
    user = "my-org"
    repo = "my_app"
    ```

    For a custom URL, use `type = "custom"` with an `href` field:

    ```toml
    [repository]
    type = "custom"
    href = "https://git.example.com/my_app"
    ```
  </Accordion>

  <Accordion title="links">
    An array of labelled URLs shown in generated documentation.

    ```toml
    [[links]]
    title = "Website"
    href = "https://example.com"

    [[links]]
    title = "Changelog"
    href = "https://example.com/changelog"
    ```
  </Accordion>

  <Accordion title="[documentation]">
    Configuration for generated HTML documentation. The `pages` field lists additional Markdown files to include.

    ```toml
    [documentation]
    pages = [
      { title = "Getting started", path = "getting-started.html", source = "docs/getting-started.md" }
    ]
    ```
  </Accordion>

  <Accordion title="[erlang]">
    Erlang/OTP-specific settings.

    | Field | Type | Description |
    |---|---|---|
    | `extra_applications` | array of strings | OTP applications to start before your application (e.g. `["inets", "ssl"]`) |
    | `application_start_module` | string | Module to call on application start |

    ```toml
    [erlang]
    extra_applications = ["inets", "ssl"]
    application_start_module = "my_app"
    ```
  </Accordion>

  <Accordion title="[javascript]">
    JavaScript-specific settings.

    | Field | Type | Description |
    |---|---|---|
    | `typescript_declarations` | boolean | Emit `.d.ts` declaration files alongside compiled output |
    | `runtime` | string | Default JS runtime: `"node"`, `"deno"`, or `"bun"` |

    ```toml
    [javascript]
    typescript_declarations = true
    runtime = "node"
    ```
  </Accordion>
</AccordionGroup>
